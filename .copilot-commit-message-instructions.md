# Conventional Commits (summary)

## Header  

`<type>(<scope>): <description>`

* **type** — one of `feat|fix|docs|style|refactor|perf|test|build|ci|chore|revert` (lower-case)
* **scope** — **REQUIRED** — pick a concise module or area (e.g., `kennitala`, `tests`, `ci`, `docs`)
* **description** — start with a lower-case verb, no final period
* **max 72 characters** for the entire header
* **lower-case only** — do not capitalise the first word of the subject
* When several change kinds are included, **pick the dominant type for the header** and list the other changes in the body:

```commit
feat(kennitala): add pagination support

* fix: correct off-by-one in limit check
* style: tidy variable names
```

## Body

* Begin the body **after one blank line**.
* Bullet points (`*`) are encouraged for clarity.
* No enforced line-length limit (rule `body-max-line-length` is disabled), but aim for ~72 characters per line for readability.

## Footer

* Begin the footer **after one blank line**.
* No line-length limit (`footer-max-line-length` is disabled).

## Lint rule summary (from `.commitlintrc.json`)

| Rule | Effect |
|------|--------|
| `type-empty`           | **Required** — header must contain a type. |
| `type-case`            | Type must be **lower-case**. |
| `type-enum`            | Type must be one of: `feat|fix|docs|style|refactor|perf|test|build|ci|chore|revert`. |
| `scope-empty`          | **Required** — scope cannot be empty. |
| `scope-case`           | Scope must be **lower-case**. |
| `subject-empty`        | **Required** — description cannot be empty. |
| `subject-full-stop`    | Do **not** end the description with a period (`.`). |
| `header-max-length`    | **72 characters maximum** for the entire header. |
| `body-leading-blank`   | Leave one blank line before the body. |
| `footer-leading-blank` | Leave one blank line before the footer. |
